## Основная информация

Реализация Universo Platformo / Universo MMOOMM / Universo Kiberplano на основе Actix Web, Yew и экосистемы Rust. Этот проект представляет собой реализацию концепций из [Universo Platformo React](https://github.com/teknokomo/universo-platformo-react) на родном Rust, с фокусом на производительность, безопасность и возможности фронтенда на базе WebAssembly.

**В этом репозитории в настоящее время общественными силами создаётся Universo Platformo / Universo MMOOMM, чтобы запустить глобальную текнокомизацию и спасти человечество от окончательного порабощения и полного уничтожения путём создания специальных массовых многопользовательских виртуальных миров, таких как Universo MMOOMM, и платформы для их создания - Universo Platformo, изначально с игровой функциональностью, а затем с добавлением функциональности Киберплана.**

Universo Platformo Rust служит основой для реализации **Universo Kiberplano** - глобальной системы планирования и реализации, которая объединяет планы, задачи и ресурсы, контролируя роботов. Эта реализация на Rust приоритизирует типобезопасность, производительность и надёжность для критически важных операций.

Более подробно обо всём этом написано в "Книге Будущего" и различных других наших материалах, большинство из которых пока плохо структурированы и не на английском языке, но прямо сейчас ведётся работа по созданию новой подробной документации, которая будет представлена на многих языках.

## Обзор

Universo Platformo Rust - это проект, который переосмысливает основные концепции из Universo Platformo React, используя:

-   **Rust + WebAssembly** для высокопроизводительного, типобезопасного фронтенда (через Yew)
-   **Actix Web** для быстрого, конкурентного бэкенда
-   **Интеграцию Supabase** для многопользовательской функциональности с абстракцией для будущей поддержки СУБД
-   **Структуру монорепозитория** с использованием Cargo workspaces
-   **Архитектуру на основе пакетов** с чётким разделением фронтенда/бэкенда

Проект нацелен на создание единой платформы для разработки интерактивных 3D приложений, которые могут быть экспортированы в различные технологии, включая AR.js, PlayCanvas, Babylon.js, Three.js и A-Frame, всё на основе гарантий производительности и безопасности Rust.

## Связь с Universo Platformo React

Этот репозиторий реализует ту же концептуальную основу, что и [Universo Platformo React](https://github.com/teknokomo/universo-platformo-react), но с использованием технологического стека Rust:

**Концептуальные эквиваленты:**
-   React → Yew (фреймворк фронтенда на основе WebAssembly)
-   Express → Actix Web (высокопроизводительный веб-фреймворк)
-   PNPM workspaces → Cargo workspaces
-   JavaScript/TypeScript → Rust
-   Passport.js → Решения для аутентификации на родном Rust

**Важно:** Это НЕ прямой перенос. Мы берём проверенные концепции из реализации на React и применяем лучшие практики экосистемы Rust, избегая любых архитектурных проблем или незавершённых функций из версии на React.

## Структура проекта

**КРИТИЧЕСКИ ВАЖНО: Модульная архитектура (ОБЯЗАТЕЛЬНА)**

ВСЯ функциональность приложения ДОЛЖНА быть реализована как пакеты в директории `packages/`. Это фундаментальное архитектурное требование, которое не может быть нарушено.

**Что ДОЛЖНО быть в packages/**:
- ✅ Все реализации функций (фронтенд и бэкенд)
- ✅ Вся бизнес-логика и код приложения
- ✅ Все UI-компоненты
- ✅ Все API-эндпоинты
- ✅ Общая инфраструктура (типы, утилиты, i18n, UI-компоненты)

**Что разрешено в корне репозитория**:
- ✅ Cargo.toml (конфигурация workspace)
- ✅ rust-toolchain.toml (спецификация toolchain)
- ✅ .gitignore, .github/ (конфигурация репозитория)
- ✅ .specify/ (рабочий процесс спецификации)
- ✅ README файлы и LICENSE

**Что ЗАПРЕЩЕНО в корне репозитория**:
- ❌ Исходный код приложения (НИКАКОЙ директории src/)
- ❌ Реализации функций
- ❌ Бизнес-логика
- ❌ UI-компоненты
- ❌ API-эндпоинты

Эта модульная структура обеспечивает:
1. Пакеты могут быть извлечены в отдельные репозитории в будущем
2. Команды могут работать независимо над разными пакетами
3. Развёртывание может быть выполнено по пакетам
4. Границы кода ясны и контролируемы

```
universo-platformo-rust/
├── packages/                  # ВЕСЬ КОД ПРИЛОЖЕНИЯ ДОЛЖЕН БЫТЬ ЗДЕСЬ
│   ├── universo-types/        # Общие определения типов (ПРИОРИТЕТ: Создать первым)
│   │   └── base/
│   ├── universo-utils/        # Общие утилиты (ПРИОРИТЕТ: Создать первым)
│   │   └── base/
│   ├── universo-api-client/   # HTTP-клиент (ПРИОРИТЕТ: Создать первым)
│   │   └── base/
│   ├── universo-i18n/         # Интернационализация (ПРИОРИТЕТ: Создать первым)
│   │   └── base/
│   ├── universo-ui-components/# Общие UI-компоненты (ПРИОРИТЕТ: Создать первым)
│   │   └── base/
│   ├── clusters-frt/          # Фронтенд кластеров (Фаза 2)
│   │   └── base/              # Основная реализация Rust/Yew
│   ├── clusters-srv/          # Бэкенд кластеров (Фаза 2)
│   │   └── base/              # Основная реализация Rust/Actix
│   ├── [feature]-frt/         # Другие пакеты фронтенда
│   └── [feature]-srv/         # Другие пакеты бэкенда
├── specs/                     # Спецификации функций
│   └── 001-initialize-rust-platformo/
├── .specify/                  # Инструменты рабочего процесса спецификации
│   ├── memory/                # Конституция и память проекта
│   ├── scripts/               # Скрипты автоматизации
│   └── templates/             # Шаблоны документов
├── .github/
│   └── instructions/          # Руководства по разработке
│       ├── github-issues.md   # Стандарты создания Issue
│       ├── github-labels.md   # Соглашения по меткам
│       ├── github-pr.md       # Руководства по Pull Request
│       └── i18n-docs.md       # Правила двуязычной документации
├── Cargo.toml                 # Конфигурация workspace (БЕЗ кода приложения)
└── README.md                  # Этот файл
```

Эта структура позволяет:

-   **Обязательная модульность**: Каждая функция ДОЛЖНА содержаться в собственном пакете(ах)
-   **Чёткое разделение**: Пакеты фронтенда и бэкенда различны (суффиксы -frt/-srv)
-   **Масштабируемость**: Новые функции ДОЛЖНЫ быть добавлены как новые пакеты в packages/
-   **Извлечение в отдельные репозитории**: Пакеты спроектированы для будущего извлечения
-   **Перспективность**: Подкаталоги `base/` позволяют альтернативные реализации

## Ключевые функции

### Паттерн трёх сущностей

Многие функции в Universo Platformo следуют паттерну иерархии из трёх сущностей:

-   **Кластеры**: Кластеры / Домены / Ресурсы
-   **Метавселенные**: Метавселенные / Секции / Сущности
-   **Уники**: (Расширенная структура сущностей)

Этот паттерн предоставляет последовательный подход к организации иерархических данных и отношений в разных функциональных областях.

### Пространства и Холсты

Будущая разработка будет включать функциональность Пространств/Холстов для создания визуальных рабочих процессов на основе узлов:

-   Интеграция узлов графа LangChain
-   Узлы UPDL (Universal Platform Description Language)
-   Визуальный редактор рабочих процессов
-   Возможности кросс-платформенного экспорта

## Рабочий процесс разработки

1. **Фаза планирования:**
   - Создать GitHub Issue (следуя `.github/instructions/github-issues.md`)
   - Написать спецификацию функции в `specs/[###-feature-name]/spec.md`
   - Получить одобрение спецификации

2. **Фаза реализации:**
   - Создать ветку функции с именем `###-feature-name`
   - Реализовать согласно спецификации
   - Писать тесты вместе с кодом
   - Обновить документацию (сначала английскую, затем русскую)

3. **Контрольные точки качества:**
   - Весь код должен проходить `cargo clippy` без предупреждений
   - Весь код должен быть отформатирован с `rustfmt`
   - Все тесты должны проходить с `cargo test`
   - Документация должна быть двуязычной с идентичной структурой

4. **Проверка и слияние:**
   - Создать Pull Request (следуя `.github/instructions/github-pr.md`)
   - PR должен ссылаться на соответствующий Issue
   - PR должен включать описания на английском и русском языках
   - PR должен пройти все проверки CI перед слиянием

## Начало работы

### Предварительные требования

-   Rust (stable) - Установить через [rustup](https://rustup.rs/)
-   Cargo (поставляется с Rust)
-   Node.js (для некоторых инструментов сборки)
-   Аккаунт Supabase (для функций базы данных)

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/teknokomo/universo-platformo-rust.git
cd universo-platformo-rust

# Собрать все пакеты
cargo build

# Запустить тесты
cargo test

# Запустить с горячей перезагрузкой во время разработки
cargo watch -x run
```

### Создание нового пакета функции

При создании новой функции, требующей фронтенд и бэкенд:

1. Создать два пакета в `packages/`:
   ```bash
   mkdir -p packages/myfeature-frt/base
   mkdir -p packages/myfeature-srv/base
   ```

2. Добавить их в workspace в `Cargo.toml`

3. Разместить ваш код Rust в соответствующих каталогах `base/`

4. Следовать подходу specification-first, документированному в `.specify/`

## Документация

Вся документация должна быть предоставлена на английском и русском языках:
